Ключевое слово `volatile` (изменчивый, непостоянный). Его необходимо использовать для переменных,
 которые используются разными потоками. Это связано с тем, что значение переменной, объявленное без `volatile`
 может кэшироваться отдельно для каждого потока, и значение из этого кэша может различаться для каждого из них.
 Объявление переменной с ключевым словом `volatile` отключает для нее такое кэширование и все запросы к переменной
 будут направляться непосредственно в память.

Non-atomic operation on volatile field:
    `volatile`: каждая индивидуальная операция чтения/записи переменной становится атомарно видимой для всех потоков.
    Однако операции, которые требуют более одной операции read/write, такие как i++(i = i + 1), которая делает одно
    чтение и одну запись, не являются атомарными, поскольку другой поток может записать в переменную i между чтением и
    записью. (Использовать AtomicInteger)
